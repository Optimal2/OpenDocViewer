OpenDocViewer manual

OpenDocViewer is used to open, review, compare, select, and print documents in a web browser. This manual describes the standard features. A deployment may have its own fields, print reasons, help text, and rules.

Basic workflow

  1. Wait until the documents have finished loading. Printing and some document actions are disabled while loading is in progress.
  2. Use the thumbnail page list to move between pages.
  3. Use the toolbar for zoom, fitting, rotation, comparison, selection, metadata, and printing.
  4. Always verify that the correct document and pages are selected before printing or saving a PDF.

Navigation

You can change page using the toolbar page controls, thumbnails, the mouse wheel, and the keyboard. The page counter shows the current page in the loaded session.

  • Previous/next page: use the toolbar arrow buttons or the keyboard arrow keys.
  • Page Up/Page Down: normally moves one page backward or forward.
  • Home/End: can be used to move to the beginning or end if the browser gives focus to the viewer area.
  • Thumbnails: click a thumbnail to open the corresponding page.

Keyboard shortcuts are ignored in text fields, lists, and other input fields so that typing does not accidentally control the viewer.

Zoom and viewing

The toolbar contains common zoom and fitting modes. Exact names may differ slightly between deployments.

  • Fit to page: shows the whole page in the available area.
  • Fit to width: makes the page as wide as the viewer area.
  • Actual size: shows the page without fit scaling.
  • Zoom in/out: changes the viewing scale step by step.

Zoom affects how the page is displayed in OpenDocViewer. Normal printing uses the print flow, not the exact current zoom level.

Thumbnails and page list

The thumbnail panel shows the pages in the session. If multiple documents are included, the page list may show document boundaries or document information depending on deployment configuration.

  • Select a page by clicking its thumbnail.
  • Use thumbnails for a quick overview of longer sessions.
  • For very large sessions, some thumbnails may load only when needed.

Documents and metadata

OpenDocViewer can display metadata from the documents in the session. Examples may include patient ID, name, unit, document date, page count, or document ID. The available metadata depends on the system that opens OpenDocViewer.

  • Page metadata: shows metadata for a specific page or document part.
  • Metadata matrix: shows metadata for multiple documents in the same session.
  • Print metadata: some metadata can also be used in print headers and footers.

If a field is missing, it is normally not shown in the printout when the template is configured with conditional fields.

Selection

If document selection is enabled, you can choose which documents or pages should be included in workflows that support selection. Selection can affect printing of selected pages.

  • Verify the selection before printing.
  • Selection may be document-based or page-based depending on deployment.
  • If no specific selection is used, the whole session or the print dialog choice normally applies.

Comparison

Comparison mode is used to show two pages or document parts side by side. It can be useful for manual review of versions, copies, or related documents.

  • Enable comparison from the toolbar if the feature is available.
  • Select the primary page and the comparison page.
  • Printing an active comparison may be handled differently from printing all pages.

Image adjustments and active page

Some deployments may allow visual adjustment of an active page, such as rotation or other viewing adjustments. Such adjustments normally belong to the active-page flow and should be checked carefully before printing.

Printing the active page may therefore differ from printing all pages, because the active page may need to use the current visual surface instead of the original image.

Printing

Open the print dialog from the toolbar print button. The dialog may contain fields for print reason, recipient/for whom, watermark, and page choice depending on deployment.

Page choice

  • Active page: prints the page currently active in the viewer. This is best when the current page has special visual handling.
  • All pages: prints the whole session or the current selection.
  • Simple range: prints one continuous page range.
  • Custom pages: prints a specified list, for example 1,3,5-8.

Direct print via HTML

Print via HTML uses the browser's normal print flow for OpenDocViewer's prepared print view. The browser's portrait or landscape choice normally applies to the whole print job.

This mode is simple and direct, but the browser print preview can be slow for many large image pages.

Print via PDF

Print via PDF makes OpenDocViewer create a PDF file from the print job first and then sends that PDF to the browser for printing. During PDF generation, progress shows how many pages are done.

In PDF mode, each page can automatically get portrait or landscape format based on the page image. This can therefore differ from direct print where the browser's format choice applies more generally.

Save PDF

If the deployment allows it, the Save PDF button is shown. It creates the same type of PDF as PDF printing, but downloads the file instead of opening print.

Copy watermark

If the deployment allows it, you can select Add copy watermark. A clear but transparent marker is then added to the printout. The deployment controls the exact text, language, and appearance.

Header and footer

Printouts can have a header and footer. These may contain date, time, print reason, user, page number, document ID, and metadata. Fields without values can be hidden automatically when the template is configured that way.

The page number in the print header normally means the page's position in the print job. The document's own page number can be shown separately in the footer.

Performance and large documents

OpenDocViewer tries to balance performance and memory use. On powerful computers, more pages and thumbnails can be prepared in advance. On computers with less memory, the viewer may be more conservative.

  • Wait until loading is complete before starting large print jobs.
  • PDF printing can provide clearer progress feedback for large print jobs.
  • The browser's own print preview can still take time, especially in Edge/Chromium.

Troubleshooting

  • The print button is disabled: wait until the documents have finished loading.
  • The print preview is slow: try PDF printing if the feature is available.
  • The wrong pages are printed: check page choice, range, and selection in the dialog.
  • Metadata is missing in print: verify that the metadata exists in the session and that the deployment template uses the correct field.
  • The manual lacks local information: the deployment can add its own manual file under help/site/.

Local customization

This default manual can be replaced per deployment without rebuilding OpenDocViewer. Place a local manual under help/site/manual.en.html and the corresponding Swedish version under help/site/manual.sv.html.
