# OpenDocViewer v2.5.1

OpenDocViewer v2.5.1 is a security patch release. All deployments should upgrade from v2.5.0 or earlier when practical.

## Security

- Forces the transitive `shell-quote` development dependency to `1.8.4`, addressing GHSA-w7jw-789q-3m8p / CVE-2026-9277 where `quote()` could leave line terminators in object `.op` values unescaped.
- The vulnerable package is pulled in through development tooling (`concurrently`) and is not part of the browser runtime bundle, but keeping the locked dependency patched avoids unsafe local, CI, and release-tooling exposure.

## Viewer Fixes

- Fixed PDF resolution boost handling so a successfully re-rendered page must replace the visible asset URL before the page is marked as boosted.
- Fixed fitted-page transitions after PDF resolution boosts so the hidden preloader treats a changed page URL as a real asset transition.
- Improved print/progress dialog text contrast, primary action hover contrast, and focus outlines for compact dialog buttons.

## Build And Packaging

- Made the Vite build identifier deterministic by default, while still allowing `ODV_BUILD_STAMP` or `ODV_BUILD_ID` to override it for release and diagnostics workflows.
- Added and hardened OMP universal-package helper scripts and command-wrapper validation tooling.
- Updated the OMP component manifest to the current artifact-only component version used by OMP packaging.
