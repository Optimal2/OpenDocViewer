# OpenDocViewer v2.5.0

OpenDocViewer v2.5.0 focuses on large-session loading performance, host-prepared source transport, and clearer diagnostics for production support.

## Highlights

- Adds source-pack stream loading for host-prepared document sessions. This lets a host provide many source files through one ordered stream instead of making the browser coordinate hundreds or thousands of individual source fetches.
- Improves large raster and TIFF sessions by keeping performance-mode memory behavior viable for higher page counts and avoiding unnecessary first-load thumbnail fallback.
- Tunes automatic worker allocation for modern browsers, with more aggressive raster worker usage in Chromium-family browsers and a conservative cap for Firefox.
- Adds richer performance overlay diagnostics for bootstrap timing, source-pack state, route counts, page-count hints, worker queue/run timing, loader phases, and copied support snapshots.

## Loading And Transport

- Added support for the `ODVSP1` source-pack stream format.
- Decoupled source-pack frame reading from frame processing so the stream can keep moving while source blobs are validated and queued.
- Added explicit `bundleUrl` startup precedence so prepared session payloads win over same-origin parent-page bootstrap data.
- Added support for long IIS session query strings in the static hosting configuration.
- Added trusted page-count hints from host-prepared bundles so the viewer can show stable total-page progress earlier.

## Rendering And Diagnostics

- Added browser labels, worker queue timing, worker run timing, and source-pack transport details to the performance overlay.
- Improved PDF worker routing with hardware-aware page policies and partitioned warm-up support.
- Added stronger page-list integrity diagnostics for duplicate, missing, ordering, and index issues.
- Added safer handling around worker task identifiers, timer APIs, memory-pressure threshold ordering, source validation metadata, and JSDoc parsing.

## Notes For Integrators

- Source-pack streaming is the preferred bulk source transport for host-prepared sessions with many small files or high-latency client connections.
- Existing Portable Document Bundle startup modes remain supported.
- Host-specific authorization, audit, temporary URL issuance, and business workflow state should remain outside OpenDocViewer.
