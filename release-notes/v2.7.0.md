## Changes Since v2.6.6

- Added a default frame-embedding policy so the viewer ships with `X-Frame-Options` and a Content-Security-Policy `frame-ancestors` directive enabled by default, reducing clickjacking and unwanted framing exposure out of the box.
- Hardened the optional system-log token handling to fail closed and keep placeholder tokens disabled, so a misconfigured or placeholder backend log token cannot silently enable log forwarding.
- Updated the dependency baseline and resolved security advisories, including brace-expansion CVE-2026-13149 and additional transitive dependency pins via overrides, and moved the test runner to the Vitest 4 line.
- Ran the Vitest suite in CI and release workflows and added component-version and local-CI validation plus tracked pre-commit and pre-push Git hooks.
- Hardened the OMP seed SQL so artifact-pointer writes are guarded and artifact ids resolve from existing rows instead of hardcoded values, and fixed the OMP repository-object build to strip runtime site configuration (`odv.site.config.js`) from artifact payloads.
- Refreshed source and agent documentation and regenerated the committed AI-agent documentation packet so CI validation sees the mapping files as current.

## Upgrade Guidance

Upgrade v2.6 deployments to v2.7.0. Earlier OpenDocViewer releases are superseded by the current framing, log-token, dependency, and OMP build/seed hardening baseline.
